Most people interact with AI as a question-answering machine. You ask, it answers. Useful — but fundamentally passive.

AI agents are different. They *do things*.

## The Shift from Tool to Actor

A traditional LLM call looks like this:

```
input → model → output
```

An agent loop looks like this:

```
goal → plan → action → observe → plan → action → ... → done
```

The model isn't just generating text. It's deciding what to do next based on what happened last. That's a fundamentally different capability.

## What Makes This Interesting to Build

The gap between "a chatbot that can call tools" and "an agent that reliably completes complex tasks" is enormous — and almost entirely unsolved.

That gap is the most interesting engineering problem I've encountered. It involves:

- **Context management** — what does the agent need to know, and when?
- **Tool design** — how do you write tools that an LLM can use reliably?
- **Failure recovery** — what happens when the model makes a wrong assumption?
- **Evaluation** — how do you know if the agent is doing the right thing?

## Where I'm Focusing

I'm building agents that automate real workflows I use daily — task management, finance analysis, development decisions.

Each one teaches me something about the gap between "it works in demos" and "it works when you're not watching."

That's the gap worth closing.
